1.  lab_resource_code
This single-file C# program fully implements Banker’s Algorithm as requested by the lab task. The program accepts user input for the number of processes and resource types, then reads allocation matrix, maximum demand matrix and available resource values. It automatically calculates the Need matrix by subtracting allocated resources from each process’s maximum requirement. Using iterative simulation, the algorithm searches for a complete safe execution sequence. Finally, it prints the safe process order if the system is safe, or warns that the state is unsafe with potential deadlock risk. All variables and logic are clearly named for readability.
2.  lab_resource_run_output
This screenshot captures the full runtime input and output of the Banker’s simulator using the standard test case provided in the lab worksheet. After typing in the allocation table, maximum demand table and available resource values, the program computes and displays the safe sequence P2 → P4 → P5 → P1 → P3 and confirms the system is in a safe state. It demonstrates the core logic of deadlock avoidance: processes that finish will release their held resources back to the shared pool to allow other waiting processes to complete their tasks.
3.  lab_resource_manual_calc
This document contains step-by-step manual calculations for Task 1, Task 2 and Task 3 with three different available resource configurations. I calculated the Need matrix for every process, checked whether current free resources can satisfy each process’s remaining demand, identified valid safe sequences, and wrote reflections about how increasing total available resources changes system performance and resource utilisation. This manual working demonstrates the theoretical calculation process of Banker’s Algorithm without code simulation.
